# Kelivo × Claude Code 订阅直连 · 手机版路线(机教版附篇)

> 没有电脑、没有 iPad,**一部手机**从零跑通主教程《Kelivo接入ClaudeCode订阅-机教版》。
> 思路:你不需要终端——**Claude Code web(claude.ai/code)自带一个云端 Linux 沙盒**,
> 所有"本地电脑"的命令都让它替你跑;你只负责网页点点点 + 复制粘贴。
>
> 本文管"人这边怎么操作";技术细节全在主教程里,你不用看懂它,AI 看得懂。

---

| 东西 | 说明 |

| Claude 订阅 | Pro 可以,**Max 推荐**(这套全靠订阅额度) |

## 1. 建一个 GitHub 空仓库(2 分钟)

Claude Code web 需要挂在一个仓库上干活,内容无所谓。

1. 手机浏览器打开 github.com,登录
2. 右上角 **+** → **New repository**
3. 名字随便(如 `my-shim`),勾上 **Add a README file**,点 **Create repository**

---

## 2. 打开 Claude Code web,喂它主教程

1. 手机浏览器(或 Claude App 里的 Code 入口)打开 **claude.ai/code**,登录你的 Claude 账号
2. 首次使用会要求 **Connect GitHub** → 按提示授权 → 选刚建的仓库
3. 新建 session → 出现聊天框
4. **把主教程全文粘贴进去**,末尾加上这段话(照抄):

> 照这份文档给我搭一套 Kelivo 接 claude -p 的后端。我只有一部手机,没有电脑:
> 1)所有终端命令都在你的环境里替我执行;
> 2)需要我在网页上操作的(Zeabur 面板、登录授权),给我保姆级的逐步指引,一步一步等我确认再继续;
> 3)需要我提供的东西(域名、密码、粘贴回来的链接),明确告诉我要什么、去哪拿;
> 4)每完成一个组件,先验证通了再进下一步。

5. 之后就是对话式推进,它指哪你点哪。

**⚠️ 沙盒网络权限**:Claude Code web 的沙盒默认限制外网访问。如果 AI 说"访问不了 xxx 域名",
去 session 的设置里放开网络权限(或允许它要求的那个域名),再让它重试。

## 3. 几个"必须你亲手做"的节点(AI 替代不了的)

全程只有这几处需要你本人出手,其余都是 AI 干活:

### 3.1 Zeabur 部署代理(主教程 §2)
AI 会让你:浏览器开 zeabur.com → 模板市场搜 **CLI Proxy API** → 一键部署 → 填三个变量(域名前缀、管理密码、API key,后两个自己编好记在备忘录)。部署完把域名告诉 AI。

### 3.2 订阅 OAuth 登录(主教程 §2,最关键的一步)
AI 会在它的沙盒里下载 Linux 版 cli-proxy-api,跑 `--claude-login`,然后**给你一个链接**:

1. 复制链接,浏览器打开 → 登录你的 Claude 订阅账号 → 授权
2. 授权后页面会跳转(可能显示打不开,**没关系**)——把**跳转后地址栏里的完整网址**复制下来
3. 粘贴回 Claude Code 的聊天框,AI 拿它换出登录凭证并上传到你的云端代理

> 这一步产物是你订阅的"钥匙",AI 只会传到**你自己的** Zeabur 代理上。教程红线:只给自己用,不共享。

### 3.3 shim 部署(主教程 §3)
AI 在沙盒里写好 kelivo-shim 代码后,用 `npx zeabur` 部署需要登录你的 Zeabur——AI 会给你一个 Zeabur 授权链接,同样是:打开 → 确认 → 回来告诉它好了。之后 Zeabur 面板上设环境变量、生成域名,AI 会一项一项报给你抄。

### 3.4 Kelivo 配置(主教程 §4)
最后回到手机 Kelivo App:供应商类型选 Claude,填 shim 域名 + 你自己编的 SHIM_KEY,拉模型、勾 reasoning、开流式。两分钟。

---

## 4. 手机操作小抄

- **复制长文本**:主教程全文很长,手机上先存到备忘录再整段复制,比网页里直接选省事
- **多任务来回切**:Claude Code 聊天 ↔ Safari(Zeabur/授权页)会频繁切换,别关掉聊天页签,session 一直在
- **session 断了/关了**:重新打开 claude.ai/code 能回到原 session 继续;万一丢了,新开一个,把主教程重新喂一遍,告诉它"前面做到了哪一步"(Zeabur 上已部署的东西都还在,不会白干)
- **沙盒是一次性的**:部署完成后沙盒销毁没关系,所有东西都跑在 Zeabur 上,沙盒只是施工队

---

## 5. 手机路线专属排错

| 症状 | 解 |
|---|---|
| claude.ai / GitHub 打不开 | 科学上网没开或节点不行,先解决网络 |
| Claude Code 里 AI 说下载不了东西 | 沙盒网络权限,见 §2 的注意事项 |
| OAuth 跳转页显示"无法打开" | 正常,要的是**地址栏的网址**,复制它粘回聊天框 |
| AI 干到一半 session 超时 | 重开 session,喂教程 + 说明进度,继续 |
| 其余一切症状 | 见主教程 §9 排错速查,或直接把报错发给 AI |
